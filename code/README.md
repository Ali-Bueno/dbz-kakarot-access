# `code/` — native decompilation of DRAGON BALL Z KAKAROT

Reverse-engineering workspace for the accessibility mod. Goal: recover the **non-reflected C++
member offsets** (menu selection indices) that UE4SS Lua cannot read, so the hybrid mod can
close its two dead ends (battle-pause selection, field/overworld Start menu). See
[`hybrid-mod-feasibility.md`](hybrid-mod-feasibility.md) for the mod-architecture decision.

## Pipeline

1. **`AT-Win64-Shipping.exe`** — copy of the retail exe. **DRM-encrypted (SteamStub v3.1 x64)**:
   `.text` entropy 8.0, entry point in `.bind`. Not directly analyzable.
2. **Unpack** with Steamless (`D:\code\decompilers\Steamless_v3.1.0.5\Steamless.CLI.exe
   --recalcchecksum AT-Win64-Shipping.exe`) → **`AT-Win64-Shipping.exe.unpacked.exe`**
   (`.text` decrypted, entropy 6.7, `.bind` removed). This is the file Ghidra analyzes.
3. **Ghidra 12** headless analysis (`ghidra/`, project `KakarotAT`), MSVC RTTI analyzer +
   Microsoft demangler on (default for the `x86:LE:64:default:windows` spec) → real class /
   vftable names. Launched via `support/analyzeHeadless_kakarot.bat` (16 GB heap).
4. **Key finding:** the game has **no C++ RTTI for its own classes** (UE4 `/GR-`; the 901 `.?AV`
   descriptors are all third-party — ICU/OpenEXR/libwebm). So there is no automatic class→vtable
   map. The way in is **UE's native-function registration table** (`{name, exec}` pairs):
   `ghidra/find_ufunc.java` finds a UFunction's native impl by its ASCII name; `decompile_addrs.java`
   dumps any address. (Java scripts — Ghidra 12 dropped Jython; `disable_media.java` is the
   pre-script that stops the embedded-PNG → `awt.dll` JVM crash.)
5. Read the native impls to pin the `this + 0xNNN` writes. Results + the recommended
   runtime-diffing finish are in `hybrid-mod-feasibility.md` (§ Ghidra findings).

## Files

- `binary-validation.md` — PE / toolchain / DRM validation (Rich header stripped by SteamStub;
  toolchain anchored via linker 14.13 = VS2017 v141 + RTTI present).
- `structs/ui_target_layouts.h` — byte-exact layouts of the target UI classes from the CXX
  header dump, with the **hidden tail gaps** (where the private selection indices live) marked.
- `hybrid-mod-feasibility.md` — Lua-hybrid vs C++-from-scratch decision + the `mem_bridge.dll`
  design (kept **separate from `prism_bridge.dll`** per requirement).
- `decompiled/` — Ghidra C output for the target classes (generated by `export_ui.py`).
- `ghidra/` — Ghidra project + logs + scripts (git-ignored; regenerable).

Big binaries and the Ghidra database are git-ignored (`.gitignore`).
